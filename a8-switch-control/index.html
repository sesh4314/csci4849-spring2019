Assignment 8: Switch Control

For this assignment you will create a web application with built-in switch control. In other words, your application will be able to be fully controlled with one key (or a small number of keys).

The goal of this activity is to gain some experience in adapting user interfaces to a very small number of controls.

Rather than using our old friend the dialer app, we'll use a new calculator app. We'll use some existing starter code (source files are linked below).

Your task is to design and implement the control scheme for this app, assuming that the user has access to between 1 and 6 keys (most importantly, fewer keys than the 16 needed for this app). You could design a two-button interface with one button being NEXT and the other SELECT; a three-button interface with NEXT, PREVIOUS, and SELECT; a five-button interface with four-way navigation, or something else. If you want to do something a little challenging (and earn extra points), you could create a one-button interface. For this you would probably want to use the JavaScript setInterval() function.

Then, you will run some tests using your interface and report the results.

What you should do

Step 1. Copy the calculator starter project

We will start this assignment with some code for an existing calculator app. Of course, if you really want to code your own calculator, you can, but using this code will save you a lot of time. You can download the three files here:

  • index.html
  • calculator.js
  • calculator.css

You should download these files and copy them to a new directory, or you can add them directly to your Github account.

Step 2. Design a switch-based interface

You will create a switch-based interface for the calculator app. Some examples of this kind of interface can be shown in these videos: Tecla switch demo, iOS switch demo.

You can choose how your switch interface works. You might want to try out a few different variations. Creating a 2-key, 3-key, or 5-key interface will probably be easiest. If you want to go above and beyond, you could try a single-key interface with a timer, or you could allow the user to customize their switch interface. You could also implement row-column scanning which can be more efficient for single-key use.

Step 3. Implement your switch interface

Build the switch interface in HTML / CSS / JavaScript. Fortunately, most of the functionality that you need for this is already written; you will just need to wire everything together.

The calculator.js file contains a number of functions that will help you build this interface:

  • selectNext, selectPrevious, selectUp, and selectDown allow you to move the selection cursor between the different keys.
  • clickSelectedItem allows you to actuate the highlighted key; for example, if the "7" key is highlighted, calling this function is equivalent to clicking that button.

The main thing that you will need to do is edit the function that starts with $(document).keypress(function(event) { to map the user's key presses to control the user interface using the above functions. In the sample code, you can press the a key to show a message and the b key to show a different message. These show you the general structure of the code.

You can certainly make further changes to the code if you want to do so. Also, since this interface is operated using keystrokes, you can interact with the interface using devices other than your computer's keyboard, such as by connecting a USB or Bluetooth keyboard, a game controller, or an Arduino or similar board that can emulate a keyboard.

Step 4. Plan out a mini-test

You are going to test out your switch-based interface and collect some data about it. To make this interesting, we are going to run a test between two conditions. Here are some ideas:

  • Comparing two different versions of your interface (e.g. 3-key vs 5-key)
  • Comparing two different parts of the body (e.g. hand vs. foot vs. head)
  • Different speeds (if you created a one-key scanning interface)

You will want to come up with a test protocol. For this study, that will probably consists of 5 to 10 equations that the user will input. These should probably be more complex than 2+2, and should use all of the keys on the calculator.

Step 5. Run your test and collect some data

Run your test a few (3-5) times in each condition. For example, you might run five trials completing the test equations using the back of the hand, and five using the foot. For each trial, record the time needed to complete the entire protocol, and count any errors that occur. To simulate real-world use, if the user makes an error, they should re-try until they successfully enter the equation.

Step 6. Write up your study findings in the "About this app" section

Fill out the section marked "About this app" in the sample HTML file. This should include:

  • Any instructions that are needed to operate the interface;
  • A description of your test. What conditions did you test? What were the example problems?
  • A description of the results. What were the average times and number of errors for each of the two conditions? What can we conclude from these results?
