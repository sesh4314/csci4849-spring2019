This demonstrates several ways to make the phone dialer application act more like a responsive app. These examples were partly inspired by this tutorial from Marksheet.io.

Note that there is no universally correct way to make a web page responsive; what you do depends on how you want your web page to work. Here are a few techniques demonstrated here. Note that all of the "responsive" behavior came about through changes to the CSS only. There are a few minor changes to the HTML, and one change to the JS (to add this new tab and select it by default).

Changes to the HTML structure:

  • Added the "About Demo" tab and content
  • Added line break tags ("br") after the form labels to make them a little easier to read
  • Moved the number pad numbers into one div. Previously each row of three buttons was in its own div. However, that means that the buttons can't reflow to be two buttons wide, or four buttons wide, etc. Now they show by default at three buttons wide but can show more or fewer as the page width changes. This is a great example of where there's not an obvious "right" answer - users may be familiar with the traditional 3 x 4 keypad, but we may want to fill the available screen space, especially on small screens.
  • Added a placeholder image to contacts in the contact list.

All of the other changes are in the CSS file and are summarized here:

  • Changed the number pad buttons to be a percentage width, with a minimum width attribute. Now they will stretch to fill the available space; when they reach their minimum size, only then will they break onto multiple lines.
  • Added a "wide screen" view for screen widths > 800px. In this mode, the tab list and content are moved to the left side using the CSS "float: left" property. In this mode, the contact icons are shown - they are hidden in the other modes.
  • Added a narrow view for screens < 550px. This one is a little bit tricky - here the CSS overrides the "display: none" attribute for each of the tab content containers, by using the "!important" attribute. It's not obvious unless you look at the JQuery documentation, but the JQuery hide() function uses display: none to hide the tabs. For this narrow CSS mode, we unhide all the tabs so that the user can just scroll down to see all of the tab content. Since we are no longer using the tab bar, we hide it for this mode.
