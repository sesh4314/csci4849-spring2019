Assignment 5: Screen Reader Test Drive

Due Wednesday March 1.

This week, we will begin to get practice using real accessible technologies! In particular, this week we will get started using screen readers.

Screen reader software is primarily used by blind and visually impaired people. Other people (such as people with dyslexia) sometimes use speech output, but a screen reader provides a complete input and output solution that uses voice only. When we follow good, accessible design practices, that increases the likelihood that our web pages will work with a screen reader. But if you want to design things that work well with a screen reader, you really need to gain some experience actually using a screen reader.

In this assignment, you will test your page from assignment 4, and another web page, with a screen reader. You will document your experience with a video or screen recording, and will update your assignment 4 to improve the screen reader user experience.

Detailed instructions

1. Set up a screen reader

You will install a screen reader on your computing device. You have some choice, depending on your computing platform.

  • On the Mac, you can use the built-in screen reader, VoiceOver.
  • On Windows, the most popular screen reader is JAWS. Although JAWS is commercial software, you can download a trial that will run for 40 minutes at a time before needing to be reset.
  • NVDA is an open source screen reader on Windows. It is less popular than JAWS but gaining in popularity.

If you do not have access to a PC or Mac, talk to Shaun. Other options include VoiceOver on iOS devices and ChromeVox for Google Chrome, but these will be more difficult to start with.

2. Practice your screen reader

Using the links above, walk through the tutorial for VoiceOver, JAWS, or NVDA until you feel comfortable using the software. Most importantly, you need to be able to use the screen reader with the screen off. You might want to practice first with the screen on, then switch the screen off.

3. Record yourself using the screen reader

You will record a video tutorial using the screen reader on two sites. Think about this as an instructional video: you will use the screen reader while navigating the page, and while describing what you are doing, so that someone who watched the video would get the general sense of how the screen reader works.

The first site will be your web page from assignment 4: you will navigate the page, fill out the form, and submit the form. This will also serve as a test for your ARIA alerts. You may need to go back and make some edits to your page to make it work correctly (if so, you can edit the original assignment 4 files or create a copy, your choice).

The second site will be of your choice. You should choose some task that you typically do with a computing device, that is more than just visiting a text page. Examples would include checking web email, reading and writing a message on Facebook messenger, shopping for something on Amazon and going through the checkout process (but not buying it, if you don't want to). The task should involve both reading content on the page, and either navigating between pages or interacting with forms.

For this part of the assignment, you should record either a video or a screen recording. You should be able to record a video on your mobile device. You could also record a screen recording using QuickTime on Mac or Windows 10's screen capture tool. If you can, you should narrate your use of the screen reader via audio. If you are unable to do this using your screen recording tool, you can instead write up your narration as text.

When recording, you should describe what you are doing. After each of the two tasks, you should also comment on what happened during the test, and whether there was anything that the web site's author could change in order to improve accessibility with a screen reader.
