Assignment 6: Designing for Low Vision

In this assignment, you will get some experience using technology for low vision, and will develop a high-contrast version of your mobile web app.

Part 1: Testing the screen magnifier

Screen magnification software is commonly used by people with various types of vision impairments. While it is difficult to gain direct experience of a vision impairment that you don't personally have, using a magnifier can help you identify some of the challenges.

As in the previous assignment, you should complete some task in a web browser, this time using the magnifier. You should load up some web page and use it to perform a task. The task should be more than just loading the web page; it should be something you actually do, like reading through all the latest news stories, catching up on your Twitter timeline, sending messages on Facebook, or ordering something online. You don't need to actually post or buy anything, but you should complete all of the steps up to that point.

The magnifier you use will depend on your platform. Here are some options:

  • Windows
  • MacOS
  • iOS
  • Android

You should try out various magnification settings; try at least a 200% zoom, and turning on high contrast mode.

This time, you can either record a video of your experience or take a screenshot of your use of the magnifier. You should make a list of any challenges that you encountered while using the magnifier.

Can't use the screen magnifier? If you can't use the magnifier for whatever reason, such as if you're already using a screen reader, you can use an alternative form of the screen reader: use gesture control or mouse pointing. Take notes about the additional challenges of this approach compared to your normal navigation method.

Part 2: Interface Design for Low Vision

For this part of the assignment, you will update your phone dialer interface to support high contrast and large type. When designing web pages, it used to be a common accessibility feature to provide users with the ability to change the type size and colors. For example, the American Foundation for the Blind page features customizable font sizes.

More recently, it has been common for web pages to omit this option, and instead allow users to adjust the font size and color in their own browsers, using either browser settings or a custom style sheet. The responsive design approach focuses on creating web pages that work well at any size.

Although most mainstream web sites do not provide built-in low vision modes, app developers may wish to include a high-contrast or large-font mode in their app (even when developing apps in HTML and JavaScript). Creating this alternative UI mode is also a great way to practice CSS, as you can create a low-vision version of your application by changing only the CSS. So that is what we will do.

Step 1: Create a copy of your phone dialer project

You will be working on your original phone dialer application. You should use a version that has three tabs: dialer, address book, and add contact. It may be easier to copy the version from Assignment 2, but remove the "Test Gestures" tab.

You should make a separate Assignment6 folder with a copy of your original HTML, CSS, and JavaScript files. Make sure to leave the previous files as-is.

Step 2: Create copies of your HTML and CSS files

You should create a copy of your index.html and phone.css files, and name them something like index-large.html and phone-large.css. These files will represent your large font mode.

Most of your work will involve working on the phone-large.css file only. However, you will need to make a few changes to your HTML files.

For index.html, you should make a link to the index-large.html somewhere on your user interface. You should use the a tag to make the link.

For index-large.html, you should make a link back to index.html. This way, you can switch back and forth between the two pages. Then, you should change the HTML in this file so that it uses the stylesheet phone-large.css, instead of phone.css. Those are the only edits you need to make to your HTML code.

Note: we are using two separate HTML files that refer to different CSS files. However, it is possible to put all of this functionality into a single HTML file, and change the current styles in JavaScript. We'll discuss how to do that later, although you don't need to do this for your assignment.

Step 3: Create a low vision style sheet

Edit the phone-large.css file to make your interface have large fonts and high contrast. As an example, you can go to the American Foundation for the Blind web page, select "Change Colors", and set the document to be white text on black background and text size to 200%.

At the very least, your style sheet should have large text, and should have light text on a dark background. If you don't know how to do this in CSS already, this is a great time to learn some new skills.

For learning about CSS, I highly recommend Mozilla's CSS reference or CSS Tricks. For this assignment, you will almost certainly want to use the CSS properties font-size, color and background-color. Another site that has some good examples is Corelangs, although that site doesn't do a good job of explaining how everything works. I wouldn't recommend using W3Schools - they often come up first on Google, but they provide very little instruction of how things work, and it's too easy to copy and paste from there without knowing what you're doing.

Step 4: Test your low vision mode

Edit your CSS and HTML to test out your low vision mode. Since you have two separate CSS files, you can toggle between normal and large mode. You should make a web page layout that works well in both: if your original A2 was ugly, this is a good time to make some changes and clean it up.

Note that making changes to your CSS may cause other problems in your code - you may need to change the color of other items to keep them visible, or rearrange elements so they fit on the screen. A major goal of this assignment is to get some practice adapting your user interface to support multiple modes, so you should spend some time trying out different approaches.

Step 5: Add an "About this Project" tab

Finally, you should add a tab to your interface that documents what you did for this assignment. Describe what you learned from your testing of the magnifier, and what design features you added to your low vision mode.

Going above and beyond

If you want to add more to your project, or if you found the previous steps easy, here are some things you can add:

  • Use a single HTML, and use JQuery to switch between the two style sheets. Google can teach you how to do this
  • Make sure your app is color-blind friendly. Are you using color in your page? Make sure anything that is differentiated by color is also differentiated in some other way. You could even add more data to demonstrate this idea, like adding two types of contacts (people and companies) to your contact list
  • Rather than a single "low vision mode", add controls to your page that enable the user to toggle accessibility features like white-on-black mode, and change font size
